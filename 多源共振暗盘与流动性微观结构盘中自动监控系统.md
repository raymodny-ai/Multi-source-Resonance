# **产品需求文档 (PRD)：多源共振暗盘与流动性微观结构盘中自动监控系统**

## **1\. 项目概述**

### **1.1 项目背景与迭代说明**

在现代美股市场中，去杠杆引发的机械式清算（踩踏）是导致指数无抵抗自跌的主因。而场外暗盘（Off-Exchange / Dark Pool）作为机构资金进行大额建仓和隐蔽换手的绝对主力，其微观异动往往比公开市场（Lit Exchanges）更早发出见底或吸筹信号。  
本系统在原有“美股自主期权链计算 \+ 盘后权威 Gamma 校准 \+ 加密金丝雀多源多维验证”**的基础上，对**机构暗盘监控模块**进行全面纵深升级。系统引入免费且高可靠的暗盘平替数据源 ChartExchange 与 Stockgrid。通过**“SqueezeMetrics 官方 DIX 基线 \+ ChartExchange 场外卖空比率细分 \+ Stockgrid 累积美元净头寸曲线”的跨平台校验体系，完美保留前序所有流动性监控逻辑，构建起针对华尔街大资金暗盘吸筹的“全景雷达”。

### **1.2 核心目标**

* **盘中自主高频估算**：每隔 5\~15 分钟通过券商原始期权链，自主计算美股名义 Gamma 敞口（GEX）。  
* **加密多源共振校验**：通过 CCXT 直连、Velo Data 清算断层、Coinglass 持仓以及 CryptoQuant 杠杆率，判定高杠杆金丝雀的排雷进度。  
* **暗盘大资金全景追踪**：将官方 DIX 指标与 ChartExchange（场外卖空比 \> 45% 触发做市商动能转化）和 Stockgrid（多周期净头寸多头拐点）进行联合去噪，锁定机构盘后实质性的左侧建仓动向。

## **2\. 数据源与数据获取架构**

本系统采用混合异构设计，通过实时 API 结合盘后高精度网页/JSON 脚本，数据源总体配置如下：

| 指标大类 | 具体监测项 | 核心数据源 (Primary) | 校准/平替数据源 (Backup/Calibration) | 获取频率 |
| :---- | :---- | :---- | :---- | :---- |
| **1\. 做市商 Gamma** | 全市场总 Gamma 敞口 ($GEX$)、Flip Zone 点位、Put Wall | **Tradier / Schwab API** (盘中拉取原始期权链自主计算) | **SqueezeMetrics** (盘后官方总 GEX) **Barchart** (盘后各行权价名义 Gamma 直方图) | 盘中 15分钟/次 盘后 1次/日 |
| **2\. VIX 期限结构** | VIX 现货、VX1\! 近月期货、VX2\! 次月期货 | **Yahoo Finance API** (^VIX, VX=F) | Interactive Brokers API | 盘中 15分钟/次 |
| **3\. 加密衍生品** | BTC 永续合约资金费率、全网持仓量 (OI)、多头清算量、交易所预估杠杆率 | **CCXT 库** (直连 Binance/OKX 盘口数据) | **Coinglass API** (聚合 OI) **Velo Data API** (清算断层监控) **CryptoQuant API** (预估杠杆率 ELR) | 盘中 5分钟/次 盘中 30分钟/次 盘后 1次/日 |
| **4\. 机构与暗盘** | DIX 指标、Lit vs Off-Exchange 占比、场外卖空比例、暗盘美元净头寸曲线 | **SqueezeMetrics 接口** (官方 DIX 基线) **DBMF ETF** (盘中量化趋势影子基金) | **ChartExchange** (场外成交与卖空明细) **Stockgrid** (多周期大宗暗盘美元净头寸) | 盘中持续 (DBMF) 每日收盘后 (DIX, ChartExchange, Stockgrid) |

## **3\. 核心量化逻辑与数据处理**

### **3.1 盘中美股 Gamma 与加密多源验证（保留原方案）**

* **美股 Gamma 逻辑**：系统盘中每 15 分钟通过 Tradier API 拉取最新期权链，并在内存中进行 $GEX\_{Local}$ 加权计算，盘后通过 SqueezeMetrics 官方公布值进行修正系数 $\\alpha$ 的动态校准，防线由 Barchart 盘后 Put Wall 柱状图最终锁定。  
* **加密杠杆清洗逻辑**：CCXT 发现低费率（$\< \-0.01\\%$）后，激活 Velo Data 的分钟级清算额（Liquidations Spike）监控与 Coinglass 的总持仓（OI）变化率计算。若一小时内 OI 断崖式下跌 $\> 15\\%$ 且大单强平消退，配合 CryptoQuant 的预估杠杆率（ELR）高位回落，判定加密市场去杠杆完成。

### **3.2 机构暗盘“三驾马车”多源验证（新增强化逻辑）**

为了彻底规避单一商业数据源（如 SqueezeMetrics）由于黑盒算法或标签微调带来的失效风险，暗盘模块的判定逻辑重构为多维相互印证系统：

* **维度一：SqueezeMetrics 官方 DIX 基线**  
  * 每日收盘后自动获取 DIX。若 DIX $\> 45\\%$，系统标记大资金在暗盘的潜在买入意向。  
* **维度二：ChartExchange 场外卖空比对冲校验（逻辑平替与微观证实）**  
  * **逻辑原理**：基于做市商对冲原理，当散户恐慌抛售股票时，暗盘做市商在场外承接买入。做市商为了对冲这部分多头头寸，会同时在场外建立空头敞口。因此，**Off-Exchange Short Volume %（场外卖空比例）的大幅拉高，在微观上等同于机构在悄悄左侧接盘**。  
  * **处理逻辑**：自动解析 ChartExchange 中 SPY 和 QQQ 的 "Volume" 与 "Short Volume" 页面。**当 Off-Exchange Short Volume % 连续 2 个交易日拉高到 45% 以上时**，视为高确信度的机构被动吸筹信号。  
* **维度三：Stockgrid 大宗交易美元净头寸趋势（宏观动量证实）**  
  * **逻辑原理**：追踪大宗暗盘交易的 Net Dark Pool Position（暗盘净头寸动向）在拉长周期内的累积状态。  
  * **处理逻辑**：提取 Stockgrid 中针对 SPY/QQQ 过去 20、60、120 个交易日内的**累积美元买入/卖出倾向曲线（Cumulative Net Dollar Inflow）**。  
  * **拐点判定**：当市场阴跌创新低，但 Stockgrid 的 20 日和 60 日累积净头寸趋势线在底部形成底背离、斜率转正（拐头向上）时，系统视其为机构实质性持续净流入，与 DIX 的飙升形成长周期宏观共振。

## **4\. 盘中监控与信号触发机制**

### **4.1 信号共振矩阵 (Alert Matrix)**

系统判定模型采用分值累计制（满分 5 分，触发门槛为 3.5 分）。暗盘板块由于引入了 ChartExchange 和 Stockgrid，判定条件更为苛刻且抗噪。

| 监测维度 | 条件 1 (左侧枯竭区) | 条件 2 (右侧确认反转) | 权重分值 | 对应整合数据源 |
| :---- | :---- | :---- | :---- | :---- |
| **1\. Gamma 状态** | 盘中本地估算 $GEX$ 负值大幅收敛 | $\\alpha \\times GEX\_{Local} \> 0$ (完全跨越翻转线) | 1.5 分 | Tradier 链计算 \+ SqueezeMetrics / Barchart 盘后校准 |
| **2\. 波动率期限** | $\\frac{VX\_{1}}{VX\_{2}} \> 1.15$ (近月极度踩踏) | $\\frac{VX\_{1}}{VX\_{2}} \< 1.0$ 且斜率向下 (回归 Contango) | 1.0 分 | Yahoo Finance 实时波动率期货流 |
| **3\. 加密杠杆清洗** | Velo 清算单分钟触顶 \+ Coinglass 总 OI 暴跌 $\> 15\\%$ | 费率重归 $\\ge 0$，CryptoQuant ELR 显著回落至安全水平 | 1.0 分 | CCXT \+ Coinglass \+ Velo Data \+ CryptoQuant 混合引擎 |
| **4\. 机构暗盘吸筹** | **满足以下三项中任意两项**： 1\. 昨日 DIX $\> 45\\%$； 2\. ChartExchange 场外卖空比 $\> 45\\%$； 3\. Stockgrid 累积美元净头寸底背离拐头。 | 盘中 DBMF ETF 日内探底并强劲反弹，收复 5 日均线 | 1.5 分 | **SqueezeMetrics \+ ChartExchange \+ Stockgrid \+ DBMF** |

## **5\. 系统功能模块与技术实现**

### **5.1 数据抓取与解析层扩展**

为了保障系统不依赖高昂的付费商业接口，针对新引入的暗盘平替源，技术实现路线规划如下：

* **ChartExchange 解析器**：  
  * **技术实现**：使用 requests 结合伪装的 User-Agent，直接请求其底层的 JSON 数据端点（通过浏览器 F12 抓包提炼其每日图表更新的底层 API，形如 https://chartexchange.com/api/v1/shortvolume/...）。  
  * **数据清洗**：重点提取 off\_exchange\_short\_volume 和 off\_exchange\_total\_volume 字段，利用 Pandas 计算比值。  
* **Stockgrid 解析器**：  
  * **技术实现**：Stockgrid 的数据多以异步动态渲染形式展现。系统模块配置 Playwright 异步无头浏览器（Headless Mode），在美股盘后固定时间（晚上 20:30）模拟访问其 Dark Pool 板块，解析 DOM 结构或直接拦截 XHR 响应获取 net\_position 数组，将滚动 20/60/120 日均线趋势写入本地 SQLite 数据库。

## **6\. 异常容错与鲁棒性设计**

针对增加了 ChartExchange 和 Stockgrid 解析器后网页抗震动要求的升级：

* **暗盘模块独立降级逻辑 (Independent Dark Pool Fallback)**：  
  * **完整运行模式**：系统盘后默认同时轮询 SqueezeMetrics、ChartExchange、Stockgrid 三方数据进行交集确认。  
  * **轻度网络故障容错**：若 ChartExchange 抓取失败而 SqueezeMetrics 运行正常，系统自动放弃“场外卖空比”的校验，将暗盘板块的判定权临时交给 DIX 基础分 \+ Stockgrid 趋势线，权重不作调减。  
  * **极端全解析失败退化（Anti-Brittle Line）**：若全网非公开数据源遭遇严厉反爬或 HTML 改版导致 SqueezeMetrics、ChartExchange、Stockgrid 同时失效，系统**启动终极保护机制**——暗盘板块得分降为 0，次日交易自动退化为依赖 **“券商 API 计算的实时 GEX \+ 盘中监控的 DBMF ETF 动量反转”** 独立支撑，并向用户发出置顶警告：\[CRITICAL\] 场外暗盘所有爬虫接口触发改版异常，已退化为纯本地实时衍生品计算流模式，请及时排查前端结构。  
* **指数退避与分布式抓取**：  
  对所有开源/免费网页解析模块强制封装 tenacity 装饰器，设定 wait\_exponential（指数退避机制），在遭遇 403 或 429 频率限制时，自动延迟 5s、15s、45s 后重试，并自动随机切换 Proxy Header 或更换请求 User-Agent 库，最大限度保障自动化脚本的盘后长期无人值守鲁棒性。
