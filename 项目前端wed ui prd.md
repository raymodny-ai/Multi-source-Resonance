## 1. 项目背景与目标

### 1.1 背景

[Multi-source-Resonance](https://github.com/raymodny-ai/Multi-source-Resonance) 是一套已完成 Phase 1–7 后端工程建设的量化监控引擎，覆盖 GEX Gamma敞口计算、VIX期限结构、加密衍生品去杠杆监控、机构暗盘三驾马车（DIX/Short Volume/Stockgrid净头寸）、多维度共振信号评分、Hawkes过程自激衰竭测算、以及基于 APScheduler 的盘中调度引擎等核心能力。当前所有分析结果仅以日志、数据库记录和 Email/Telegram/Discord 推送形式输出，缺乏可交互的可视化界面，导致交易员在复盘与盘中决策中需要直接读取原始数据。

### 1.2 前端 UI 的定位

前端 Web UI 作为后端监控系统的**可视化交互层**，不承载任何计算逻辑，仅负责：

- 实时消费后端 REST API / WebSocket 数据流并渲染图表
- 将多维度共振信号以**视觉优先级**形式呈现，减少交易员认知负荷
- 提供配置管理界面，替代直接修改 config 文件的操作


### 1.3 产品目标

| 目标编号 | 目标描述 | 衡量指标 |
| :-- | :-- | :-- |
| G1 | 盘中关键信号的感知时延 < 5 秒 | 从后端写库到前端展示的端到端延迟 |
| G2 | 使交易员无需查看日志即可完成全链路监控 | 日志查看频次降低 ≥ 80% |
| G3 | 支持移动端盘外快速告警确认 | 移动端核心功能覆盖率 ≥ 60% |
| G4 | 配置修改可通过 UI 完成，无需 SSH | 参数配置覆盖率 100% |

## 2. 用户画像与使用场景

### 2.1 用户角色

**角色 A：盘中量化交易员**

- 目标：在 NYSE/NASDAQ 交易时段（09:30–16:00 ET）实时跟踪共振信号触发，辅助仓位调整决策
- 使用设备：双屏桌面工作站
- 核心痛点：信号分散于日志与推送消息，无法快速建立多维度全景认知

**角色 B：风控/研究员（盘后复盘）**

- 目标：盘后回溯当日共振信号时间轴，分析误报/漏报，调优阈值参数
- 使用设备：笔记本电脑，偶尔使用手机
- 核心痛点：历史信号无时间轴视图，阈值修改需直接编辑配置文件

**角色 C：系统管理员**

- 目标：监控各数据源爬虫健康状态，处理降级模式告警
- 使用设备：任意设备（含手机）
- 核心痛点：系统健康状态不可视，故障发现依赖被动推送


### 2.2 核心使用场景

**场景 S1：盘中实时监控（09:30–16:00 ET）**
交易员打开仪表盘，左侧面板显示当前共振等级（Level 1/2/3），四大域指标卡实时闪烁更新，共振信号触发时顶部横幅出现橙/红色告警条，右侧暗盘趋势图滚动追加新数据点。

**场景 S2：盘后信号复盘**
研究员进入共振信号历史时间轴页，筛选当日 Level 2+ 事件，逐一查看各维度触发明细，导出 CSV 用于策略回测。

**场景 S3：异常响应**
系统管理员收到 Telegram 告警，手机打开移动端状态页，确认是哪个数据源降级，点击"强制刷新"触发后端重试。

## 4. 功能需求清单

### 4.1 多维度实时仪表盘 `P0`

**页面定位：** 系统默认首页，聚合四大核心域的关键指标，面向盘中快速决策。

#### 4.1.1 功能描述

| 组件 ID | 组件名称 | 数据来源（后端模块） | 刷新策略 |
| :-- | :-- | :-- | :-- |
| D-01 | GEX Gamma 敞口仪表盘卡 | `quant_logic/gex_calculator.py` | WebSocket 推送，≤ 5s |
| D-02 | VIX 期限结构图（3 档） | `data_fetchers/vix_fetcher.py` | 轮询，60s |
| D-03 | 加密去杠杆强度计 | `data_fetchers/crypto_fetcher.py` | WebSocket 推送，≤ 10s |
| D-04 | 暗盘综合得分卡 | `signal_engine/` | 轮询，60s |
| D-05 | 共振等级横幅（Level 1/2/3） | `signal_engine/resonance_scorer.py` | WebSocket 推送，≤ 3s |
| D-06 | Hawkes 衰竭概率进度条 | `quant_logic/hawkes_process.py` | 轮询，30s |

#### 4.1.2 GEX 敞口卡详细要求

- 显示当前 SPX/SPY 的 **净 GEX 值**（单位：十亿美元）、**Gamma Flip 价位**
- 以半圆仪表盘形式展示 GEX 量级，颜色编码：绿（正 GEX，市场稳定）→ 红（负 GEX，市场不稳）
- 悬停 Tooltip 显示：计算时间戳、使用的期权合约数量、与前日 GEX 对比变化百分比


#### 4.1.3 VIX 期限结构图详细要求

- X 轴：VIX 近月、次月、季度到期（3 个节点）
- 叠加显示：今日曲线（实线）、昨日曲线（虚线）、30 日均值（灰色参考线）
- 期限结构倒挂时（Backwardation），图形区域背景自动变为淡红色高亮


#### 4.1.4 加密去杠杆强度计详细要求

- 显示 BTC 与 ETH 的 **资金费率**（Funding Rate）、**24h 清算量**（百万美元）
- 去杠杆强度以 0–100 进度条表示，颜色分级：0–30 绿、30–70 黄、70–100 红
- 支持切换显示 Binance / Bybit / OKX 多交易所数据源


#### 4.1.5 验收标准

- [ ] 四大域卡片在 1920×1080 分辨率下无需滚动即可全部可见（Above the fold）
- [ ] WebSocket 断线时卡片显示"数据延迟"橙色警示标签，不展示过期数据
- [ ] 共振等级从 Level 1 → Level 2+ 时，顶部告警横幅在 3 秒内出现并播放提示音（可静音）

### 4.2 暗盘数据详情页 `P0`

**页面定位：** 专注机构暗盘三驾马车数据的深度可视化，支持盘中观察与盘后复盘。

#### 4.2.1 DIX（Dark Index）走势图

- 折线图，X 轴为日期（默认显示近 30 个交易日，可调整至 3 个月/6 个月/1 年）
- Y 轴：DIX 值（0–100%）+ 标准差区间带（灰色半透明填充）
- **底背离标注**：当 DIX 上升而标的价格下跌时，自动在图上标注绿色三角标记（▲）并附文字"潜在底背离"
- 悬停显示：具体日期、DIX 值、该日对应 SPY 收益率


#### 4.2.2 卖空比（Short Volume Ratio）图表

- 柱状图叠加折线图（Combo Chart）：柱状表示每日卖空量（百万股），折线表示 5 日移动均值
- 颜色编码：卖空比 > 50% 时柱状变为深红色
- 右侧 Y 轴显示卖空占总成交量的百分比
- 支持按个股 Ticker 筛选（默认 SPY，可切换至 QQQ、IWM 等）


#### 4.2.3 Stockgrid 净头寸趋势

- 面积图，正净头寸（净买入）填充绿色，负净头寸（净卖出）填充红色
- 叠加 SPY 价格线（次 Y 轴，灰色）以直观展示资金流与价格的背离
- **底背离标注算法**：净头寸近 N 日创新低但 SPY 价格未创新低时自动标注（N 可在配置后台调整）
- 数据时间粒度：日级（盘后更新）


#### 4.2.4 三驾马车综合评分面板

- 页面右侧固定侧边栏，显示：DIX 得分（0–3 分）、Short Volume 得分（0–3 分）、Stockgrid 得分（0–3 分）
- 合计得分 ≥ 7 时显示"机构暗盘强买入信号"绿色徽章
- 历史得分热力图（Calendar Heatmap）：过去 90 天每日综合得分，颜色深浅表示强弱


#### 4.2.5 验收标准

- [ ] 底背离标注与后端 `signal_engine` 中的判断逻辑保持一致（一致性测试用例 ≥ 10 组）
- [ ] 图表时间轴切换（30日/3月/6月/1年）响应时间 < 500ms（数据已缓存）
- [ ] 卖空比数据延迟标注（数据为 T-1 盘后数据，界面需显示"数据截至昨日收盘"）

### 4.3 共振信号面板 `P0`

**页面定位：** 核心决策支持页，展示当前共振等级、触发条件逐项状态及历史信号时间轴。

#### 4.3.1 当前共振状态仪表盘

- **等级显示**：页面中央大字展示当前等级（LEVEL 1 / LEVEL 2 / LEVEL 3），对应配色：
    - Level 1：黄色（⚠️ 预警）
    - Level 2：橙色（🔶 注意）
    - Level 3：红色（🔴 高风险）
    - 无触发：绿色（✅ 正常）
- 显示**触发时间戳**与**距上次触发时长**


#### 4.3.2 触发条件逐项点亮组件

采用 Checklist 卡片设计，每个维度单独一行，实时标注当前是否满足触发条件：


| 维度 | 触发条件描述 | 状态指示 |
| :-- | :-- | :-- |
| GEX | Gamma 敞口负值且绝对值 > 阈值 | 🔴 / ⚫ |
| VIX 期限结构 | 倒挂（Backwardation）检测 | 🔴 / ⚫ |
| DIX | 连续 N 日 DIX 超过阈值 | 🟡 / ⚫ |
| Short Volume | 卖空比 > 阈值 | 🟡 / ⚫ |
| Stockgrid | 净空头头寸 > 阈值 | 🟡 / ⚫ |
| 加密去杠杆 | 24h 清算量 > 阈值 | 🔴 / ⚫ |
| Hawkes 衰竭 | 衰竭概率 > 阈值 | 🟠 / ⚫ |

- 每行显示实际数值与当前阈值，方便判断接近程度（进度条式距离指示）
- 鼠标悬停展示该维度的详细数据与数据来源说明


#### 4.3.3 历史信号时间轴

- 纵向时间轴（Timeline），每个事件节点显示：触发时间、等级、持续时长、该时间段标的涨跌幅
- 支持按等级筛选（Level 1 / Level 2 / Level 3）、按日期范围筛选
- 点击任意事件节点，展开**信号详情抽屉**（Drawer），包含触发时所有维度的快照数据
- 支持导出选定时间范围的信号记录为 CSV


#### 4.3.4 Hawkes 过程可视化

- 折线图展示事件强度函数 λ(t) 随时间的衰减曲线
- 在图上标注历史"跳跃事件"（Jump Events）节点（黑色竖线）
- 当前衰竭概率以百分比显示，附历史分布直方图对比


#### 4.3.5 验收标准

- [ ] 共振等级变化后，触发条件逐项面板在 3 秒内同步更新
- [ ] 历史时间轴支持查看 180 天内的所有信号记录（分页加载，每页 20 条）
- [ ] 信号详情快照数据与数据库记录完全一致（数据完整性校验）
- [ ] CSV 导出功能在 5000 条记录内完成时间 < 3s

### 4.4 告警通知中心 `P1`

**页面定位：** 集中管理告警历史、通知渠道状态及免打扰配置。

#### 4.4.1 告警历史列表

- 表格展示：时间、告警等级、触发事件类型、推送渠道、状态（已送达/失败/忽略）
- 支持按时间范围、等级、渠道类型筛选与排序
- 失败告警显示红色警示标签，并提供"重新发送"操作按钮


#### 4.4.2 通知渠道状态面板

显示三个渠道的实时连通性状态，对应后端 [notification](https://github.com/raymodny-ai/Multi-source-Resonance/tree/main/notification) 模块：


| 渠道 | 状态指示 | 显示内容 |
| :-- | :-- | :-- |
| Email | 🟢/🔴 | 最后发送时间、当日发送量 |
| Telegram | 🟢/🔴 | Bot 连通性、Chat ID 配置状态 |
| Discord | 🟢/🔴 | Webhook URL 有效性、最后推送时间 |

- 点击各渠道卡片可触发**测试推送**（发送一条测试消息验证连通性）


#### 4.4.3 免打扰与冷却期配置

- 免打扰时间段设置：起止时间（HH:MM）+ 允许穿越的最低等级（如"Level 3 始终通知"）
- 冷却期（Cooldown）配置：同一信号类型的最短推送间隔（分钟），防止短时间内重复告警
- 当前冷却期状态展示：各信号类型距下次可推送的剩余时间


#### 4.4.4 验收标准

- [ ] 告警历史列表支持 10,000+ 条记录的虚拟滚动，页面不卡顿
- [ ] 测试推送在点击后 10 秒内在对应渠道收到消息
- [ ] 冷却期配置修改立即生效（无需重启后端进程）

### 4.5 系统状态监控页 `P1`

**页面定位：** 面向系统管理员，提供后端运行健康的全景视图。

#### 4.5.1 数据源连通性状态

对应 [data_fetchers](https://github.com/raymodny-ai/Multi-source-Resonance/tree/main/data_fetchers) 目录下各爬虫模块的健康状态：


| 数据源 | 类型 | 状态 | 指标 |
| :-- | :-- | :-- | :-- |
| Squeezemetrics (DIX) | 暗盘 | 🟢/🟡/🔴 | 最后成功时间、失败次数/24h |
| CBOE (VIX) | 衍生品 | 🟢/🟡/🔴 | 数据时间戳、解析成功率 |
| Binance/Bybit (加密) | 加密 | 🟢/🟡/🔴 | WebSocket 连接状态、消息速率 |
| Polygon/yfinance (股票) | 股票 | 🟢/🟡/🔴 | API 配额剩余、最后刷新时间 |
| Stockgrid | 暗盘 | 🟢/🟡/🔴 | HTML 爬取成功率 |

- 🟡 降级模式：表示该数据源使用缓存数据，需标注"降级模式·使用缓存"
- 提供"强制刷新"按钮，触发后端对应 fetcher 的手动重试


#### 4.5.2 任务调度日志

对应后端 `main_scheduler.py` 中的 APScheduler 任务：

- 表格显示各定时任务名称、计划执行时间、上次执行结果（成功/失败/超时）、执行耗时
- 支持实时 Log Streaming（WebSocket），显示后端 Python 日志流（级别：INFO/WARNING/ERROR）
- 日志支持关键字搜索与级别过滤，ERROR 级别日志以红色高亮


#### 4.5.3 系统资源指标

- CPU / 内存 / 磁盘使用率（实时折线图，滚动窗口 60 分钟）
- SQLite 数据库文件大小与最后 VACUUM 时间
- 进程运行时长（Uptime）


#### 4.5.4 验收标准

- [ ] 任一数据源状态变为 🔴 时，系统状态页顶部立即出现红色告警横幅
- [ ] 日志流每秒最多刷新一次（防止大量日志导致页面卡顿）
- [ ] 强制刷新按钮有操作确认弹窗，执行后显示"任务已提交"反馈

### 4.6 参数配置后台 `P2`

**页面定位：** 替代直接编辑 `config/` 目录文件，提供可视化配置管理。

#### 4.6.1 信号阈值调优

表单式界面，对应后端各信号维度的阈值参数：

- GEX 触发阈值（十亿美元）
- DIX 连续天数 N 与最低 DIX 值

- 各 fetcher 的轮询间隔（秒/分钟）独立配置

#### 4.6.4 验收标准

- [ ] 所有配置修改在前端点击"保存"后通过 API 持久化至后端配置文件，**不直接修改运行中进程的内存状态**

## 5. 非功能性需求

### 5.1 性能指标

| 指标 | 目标值 | 测量方法 |
| :-- | :-- | :-- |
| 首屏加载时间（FCP） | < 2.0s（缓存命中） | Lighthouse / WebPageTest |
| 仪表盘数据刷新延迟 | < 5s（WebSocket 推送） | 前端 timestamp diff |
| 图表交互响应时间（缩放/筛选） | < 300ms | Chrome DevTools |
| API 请求超时阈值 | 10s（前端侧设定） | Axios/Fetch timeout 配置 |
| 前端 JS Bundle 大小 | < 800KB（Gzip） | Webpack Bundle Analyzer |

### 5.2 浏览器兼容性

- **必须支持：** Chrome ≥ 110、Edge ≥ 110、Safari ≥ 16（macOS/iOS）
- **建议支持：** Firefox ≥ 110
- **不要求支持：** IE 系列


### 5.3 安全性

- **API 鉴权：** 后端 API 需实现 JWT Bearer Token 鉴权，前端登录后获取 token 并存储于内存（不存 localStorage 防止 XSS）
- **敏感配置传输：** 所有包含 Token/Password 的 API 调用强制走 HTTPS
- **操作审计：** 参数配置修改、强制刷新等写操作记录操作日志（操作者、时间、变更内容）
- **CORS：** 后端 API 服务器限制 Origin 白名单，仅允许前端部署域名访问


### 5.4 响应式适配

- **桌面端（≥ 1440px）：** 完整功能，多列布局，图表交互全功能
- **平板端（768px–1439px）：** 核心功能可用，图表简化（隐藏次要曲线）
- **移动端（< 768px）：** 仅展示仪表盘总览卡片、共振等级、告警历史列表；图表折叠为可展开式卡片

## 6. 技术栈建议

### 6.1 前端框架

**推荐：React 18 + TypeScript**

- 理由：生态成熟，与 Python 后端解耦彻底，WebSocket 集成方案（`react-use-websocket`）完善；大量量化仪表盘开源模板可复用
- 状态管理：**Zustand**（轻量，优于 Redux；适合实时数据流状态）
- 路由：**React Router v6**

备选：Vue 3 + Vite（构建速度更快，适合团队已有 Vue 经验的场景）

### 6.2 图表库

| 用途 | 推荐库 | 理由 |
| :-- | :-- | :-- |
| 折线图、面积图、柱状图 | **TradingView Lightweight Charts** | 专为金融时序设计，性能优异，支持十万级数据点 |
| 仪表盘半圆图、进度条 | **Apache ECharts** | 仪表盘 Gauge 组件原生支持，样式高度可定制 |
| 热力图（Calendar Heatmap） | **Apache ECharts** | 内置 Calendar + HeatMap 组合 |
| 时间轴（Timeline） | **vis-timeline** 或 自定义 CSS Timeline | 金融事件时间轴展示 |
| 系统状态折线图 | **Recharts** | React 生态原生，API 简洁 |

### 6.3 UI 组件库

**推荐：Ant Design 5.x**

- 提供完整的 Table、Form、Drawer、Modal 等企业级组件
- 深色主题（`theme: { algorithm: theme.darkAlgorithm }`）适合金融终端风格
- 替代方案：Shadcn/ui（更轻量，适合定制度更高的设计系统）


### 6.4 与现有 Python 后端的 API 交互方案

**后端需新增 FastAPI 服务层**（当前后端无 HTTP 服务）：

```
Python 后端架构扩展方案：

现有后端                   新增 API 层                前端
data_fetchers/    →      FastAPI + Uvicorn    →    React 18
signal_engine/    →      REST API + WebSocket →    Zustand Store
notification/     →      SQLite ↔ API        →    TradingView Charts
main_scheduler.py →      JWT Auth            →    Ant Design UI
```

**具体接口策略：**

- **REST API（FastAPI）：** 用于历史数据查询、配置读写、告警历史查询
- **WebSocket：** 用于实时推送共振等级变化、GEX 更新、系统状态变化
- **推荐库：** `fastapi`, `uvicorn`, `websockets`, `python-jose`（JWT）

## 8. UI/UX 设计原则

### 8.1 视觉主题

采用**深色金融终端风格**（Dark Theme），主色调参考 Bloomberg Terminal / TradingView Dark：

- 背景：`#0D1117`（GitHub Dark 风格）
- 卡片背景：`#161B22`
- 边框：`#30363D`
- 主强调色：`#58A6FF`（蓝）
- 告警色：`#F85149`（红）、`#D29922`（黄）、`#3FB950`（绿）


### 8.2 信息密度控制

- **仪表盘**：每屏信息密度控制在 6–8 个核心指标，避免 "数字墙" 效应
- **详情页**：默认折叠次要指标，用户主动展开
- 使用**进度条 + 颜色**代替原始数值（原始数值通过 Tooltip 访问）


### 8.3 关键告警视觉优先级

层级由高到低：

1. **全屏遮罩（Level 3）**：屏幕边缘闪烁红色 Border + 顶部固定红色横幅 + 可选音效
2. **顶部横幅（Level 2）**：固定橙色横幅，不遮挡操作区域
3. **通知角标（Level 1）**：导航栏铃铛图标角标，不打断当前操作

### 8.4 桌面端与移动端差异策略

| 功能 | 桌面端 | 移动端 |
| :-- | :-- | :-- |
| 仪表盘 | 4列网格全展示 | 单列卡片，默认只展示共振等级 |
| 图表 | 完整交互（缩放/高亮/Tooltip） | 静态截图式，点击跳转详情 |
| 配置后台 | 完整表单 | 只读模式（移动端不建议修改配置） |
| 系统日志 | 实时流 + 搜索 | 只显示最近 20 条 ERROR 日志 |
